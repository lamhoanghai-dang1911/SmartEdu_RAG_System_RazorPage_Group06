- `testing/`: unit test and testing strategy guidelines.
- `security/`: security and secret management guidelines.

## Purpose

The purpose of this documentation is to help contributors understand how to work consistently in this project.

These documents help the team:

- Keep code style consistent.
- Improve code readability.
- Reduce review time.
